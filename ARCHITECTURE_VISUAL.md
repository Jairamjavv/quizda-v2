## ğŸ“ Key Principles Applied

### 1. Single Responsibility Principle

Each layer has **one** clear purpose:

- Components â†’ Render UI
- Hooks â†’ Manage state
- Services â†’ Business logic
- Repositories â†’ Database access

### 2. Dependency Inversion

High-level modules don't depend on low-level modules:

```tsx
// Service depends on abstraction (Repository interface)
class QuizService {
  constructor(private repo: QuizRepository) {}
}

// Not on concrete implementation
class QuizService {
  constructor(private db: Database) {} // âŒ
}
```

### 3. Interface Segregation

DTOs define minimal contracts:

```tsx
// Only expose what's needed
interface UserDTO {
  id: number;
  email: string;
  // NOT: passwordHash, internalFlags, etc.
}
```

### 4. Open/Closed Principle

Open for extension, closed for modification:

```tsx
// Easy to add new storage without changing service
interface StorageService {
  getItem<T>(key: string): T;
}

class LocalStorageService implements StorageService {}
class IndexedDBService implements StorageService {}
```
